The domain

Clinician encounter→ Prescription state machine→ Order + payment (idempotent)→ Pharmacy fulfillment

Every transition is guarded, optimistically locked, and written — with its audit row — inside one transaction. The transition table is shared Zod data, so the web client and the API can never disagree on a legal move.

Proof #1 — the audit log is append-only, enforced by the database

A trigger refuses UPDATE/DELETE for any role — even the table owner. A second layer revokes those privileges from the app's DB role entirely. Run live on Postgres 17:

# after inserting one audit row (id = 1):
UPDATE audit_log SET action = 'tampered' WHERE id = 1;
ERROR: 23000: audit_log is append-only: UPDATE is not permitted

DELETE FROM audit_log WHERE id = 1;
ERROR: 23000: audit_log is append-only: DELETE is not permitted

# the row is still there. the log only grows. ✓

Proof #2 — PHI is stored only as ciphertext

PHI is envelope-encrypted (AES-256-GCM, per-value data keys wrapped by a KMS master key) before it ever reaches the database. Read back from the live table:

columnstored valueis envelopeleaks plaintext
indication_encv1.L27CP0-3dtgrlibxykKPf…(200 chars)truefalse

Original plaintext was "Indication: soft-tissue recovery; BPC-157 250mcg BID x8wk". A database dump alone yields nothing — the key lives outside the DB.

Self-built auth — refresh rotation with reuse detection

// Each login starts a token family. Every refresh rotates the token.
// Replaying a spent token = theft → revoke the whole family.
if (record.revoked || record.replacedBy !== null) {
  await this.store.revokeFamily(record.familyId);
  throw new UnauthorizedError('Refresh token reuse detected — session revoked');
}

bcrypt · short-lived JWT access + rotating opaque refresh (stored only as a SHA-256 hash) · TOTP MFA (RFC 6238, from scratch) · RBAC. Login timing is uniform whether or not the email exists.

Run it yourself

git clone https://github.com/TobiasBoscoBrown/rxledger
cd rxledger && npm install
docker compose up -d db
cp .env.example .env
npm --workspace @rxledger/api run db:migrate
npm --workspace @rxledger/api run db:seed
npm test            # 49 + 6 green, 80/80/80 gate
npm run dev         # http://localhost:8787/health

Production target is ECS Fargate (Dockerfile + infra/main.tf with KMS-encrypted RDS, Redis, S3, CloudTrail, and CloudWatch-gated auto-rollback). This page is the project showcase; the system itself lives in the repo.
